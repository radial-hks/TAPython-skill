## External Json[¶](#external-json)

This guide explains how to use external JSON files in Chameleon UI.

## Overview[¶](#overview)

The Chameleon UI .json file can reference other json files for code reuse.

## ExternalJson[¶](#externaljson)

### Basic Usage[¶](#basic-usage)

```json
{
    "autoWidth": true,
    "SBox": {
        "WidthOverride": 480,
        "Content": {
            "ExternalJson": "ZeldaWorldMapEditor_Buttons.json"
        }
    }
}
```

This is equivalent to copying the content of "ZeldaWorldMapEditor_Buttons.json" directly.

## Multiple Inclusions[¶](#multiple-inclusions)

You can include the same external JSON multiple times:

```json
{
    "SBorder": {
        "Content": {
            "SVerticalBox": {
                "Slots": [
                    {"ExternalJson": "Buttons.json"},
                    {"ExternalJson": "Buttons.json"},
                    {"ExternalJson": "Buttons.json"}
                ]
            }
        }
    }
}
```

## Nested References[¶](#nested-references)

Note:
Nested references are supported, but circular references need to be avoided.

## Differentiate Button Click Events[¶](#differentiate-button-click-events)

Use `%widgetPath` to distinguish which button was clicked:

```python
def on_button_click(self):
    widget_path = data.get_var("%widgetPath")
    # Handle based on widget path
```

Note:
Now %widgetPath only supports SButton widget.

## Summary[¶](#summary)

### Pros[¶](#pros)

- Reduce complexity and size of a single json file
- Reuse UI code across multiple places

### Cons[¶](#cons)

- UI json files become less intuitive
- Widget path in console differs from PyCharm json crumb path

### Recommendation[¶](#recommendation)

Put repetitive UI code (like 16x16 map buttons) or UI code generated by scripts into external json files.
